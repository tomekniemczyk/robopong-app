Najważniejsze odkrycia

  1. Oryginał NIE ma osobnego warmup. Zamiast tego setup drilla robi pre-load piłki 2× z odstępem 1800ms — silniki kręcą się ~1.8s przed pierwszym T.
  2. Kalibracja jest 3-etapowa (3 osobne ekrany: Rotation → Height → Oscillation/Speed). Trzeci etap używa V\r × 2.
  3. Firmware NIE persystuje kalibracji — musi być wysłana po każdym połączeniu (SpeedCAL, HeightCAL, OscCAL, RotCAL w config lokalnym).
  4. Auto-detect trybu drilla: FW < 701 || USB → SYNC (B+T), FW ≥ 701 && BLE → ASYNC (wTA+A+END).
  5. Windows zawsze SYNC — FTDI USB wymusza USB_Mode=true, nigdy nie używa END.
  6. BUG w oryginale: SpeedCAL jest martwym kodem — odczytany z config ale natychmiast nadpisywany na 0 w setBall().
  7. Kompensacja opóźnień — jeśli T trwa > 625ms, następny wait skracany przez współczynnik 0.000125 × waitTime + 0.5625.
