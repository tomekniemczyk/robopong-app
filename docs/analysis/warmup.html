🔥 Warmup — rozgrzewanie silników

Kiedy i jak oryginalne aplikacje Newgy/Donic rozkręcają flywheel przed pierwszym rzutem.

⚠ Kluczowe odkrycie: Oryginalne aplikacje NIE mają dedykowanej fazy warmup. Silniki rozkręcają się "przy okazji" — podczas setup drilla wykonywany jest pre-load piłki (2× B z odstępem 1800ms), co zapewnia że flywheel kręci się ~1.8s zanim padnie pierwszy T. AcePad wprowadza własny 3s warmup jako dodatek, bo w testach 2s nie wystarczało.

2. Timeline z dokładnymi timingami

H\r t = 0ms
ClearBall — zatrzymanie silników (zabezpieczenie)
H\r t = +200ms
Drugie ClearBall — idempotentne, pewność że zatrzymane
B{first_ball}\r t = +1200ms
SetBall z parametrami pierwszej piłki — silniki zaczynają rozkręcać
B{first_ball}\r t = +1500ms
Reinforce — drugie wywołanie identyczne (cache-proof)
T\r t = +3000ms
Pierwszy rzut. Silniki kręciły ~1.8s przed tym momentem

4. Pseudokod algorytmu

# FW < 701 lub USB_Mode — SYNC (stary protokół)
def setup_drill(first_ball):
    robot.send("H\r")                              # clear #1
    wait(200)
    robot.send("H\r")                              # clear #2
    wait(1000)
    robot.send(format_B(first_ball, force=True))   # silniki start
    wait(300)
    robot.send(format_B(first_ball, force=True))   # reinforce (identyczne!)
    wait(1500)
    # od tego momentu RunDrill() — pierwszy T\r

# FW >= 701 BLE — ASYNC (nowy protokół)
def setup_drill_fw701(first_ball):
    robot.send("H\r")
    wait(200)
    robot.send("H\r")
    wait(1000)
    # brak pre-load — używany wTA+A+END, robot sam rozkręca silniki
    # (patrz: drille.html — tryb ASYNC)

7. Co robi AcePad (stan obecny)

AcePad dodaje 3s warmup (2 fazy: boosted RPM + settle) przed pierwszym rzutem — bo w testach flywheel nie osiągał obrotów w 2s.

Memory: feedback_motor_warmup.md

Jest to odstępstwo od oryginału — należy świadomie uzgodnić:

  • Czy warmup 3s jest rzeczywiście potrzebny, czy wynika z błędu w timingu setup (brak 1800ms pre-load)?
  • Jeśli 3s warmup zostaje — czy użyć boosted RPM (wyższe niż drill), czy parametrów pierwszej piłki?
  • Jak to współgra z trybem ASYNC (FW ≥ 701), gdzie robot sam zarządza silnikami?

8. Obszary do weryfikacji

  • Minimalny czas rozkręcania flywheelu — w RE brak danych o timingu RPM. Potrzebny pomiar na żywo (debug output TOP: 0\r\nBOTTOM: 838\r\n z FIRMWARE_RE.md:126 pokazuje tylko idle).
  • Czy pre-load 1800ms wystarcza — oryginał nie ma warmup, więc prawdopodobnie tak. AcePad ma krótszy setup → stąd potrzeba własnego warmup.
  • Delay 300ms między dwoma B — dlaczego dwa identyczne setBall? Cache-proof? Potwierdzenie odpowiedzi?
