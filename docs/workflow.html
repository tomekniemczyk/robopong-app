RoboPong Workflow

Architektura systemu

Frontend (Vue 3 CDN)

  • Single-file SPA (index.html)
  • WebSocket /ws (real-time)
  • REST API (CRUD)
  • Tab-based navigation (v-show)
  • i18n: PL/EN/DE/FR/ZH

Backend (FastAPI)

  • main.py — REST + WS router
  • robot.py — orkiestracja
  • training.py — state machine
  • transport.py — BLE/USB/SIM
  • recordings.py — ffmpeg
  • audio.py — WAV playback

Transport BLE

  • Bleak (BleakClient)
  • MLDP protocol (Microchip)
  • D-Bus agent (pairing)
  • bluetoothctl (scan/pair)
  • Terminator: \r

Kamera / Nagrywanie

  • motion (MJPEG, port 8081)
  • ffmpeg (MJPEG → H.264 MP4)
  • 15 fps, CRF 18, ultrafast
  • Auto-delete <30s (skip) / <3s
  • Voice notes (WebM upload)

Storage

  • SQLite: robopong.db (12 tabel, w tym calibration per-device)
  • SQLite: presets.db
  • JSON: drills, exercises, trainings
  • recordings/<player_id>/*.mp4
